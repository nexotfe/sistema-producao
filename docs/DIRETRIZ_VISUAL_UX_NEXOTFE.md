# Diretriz Visual e UX - NEXOTFE

## Principio Principal

Menos e mais.

A NEXOTFE deve parecer uma fabrica organizada digitalmente, nao um software complicado cheio de telas.

## O Que Transmitir

- Leveza
- Velocidade
- Simplicidade
- Organizacao
- Clareza industrial
- Aparencia moderna
- Foco operacional

## Evitar

- Sistema pesado
- ERP antigo
- Software burocratico
- Painel poluido
- Dashboard generico
- Cards gigantes
- Excesso de sombras
- Excesso de bordas
- Excesso de botoes
- Excesso de cores
- Excesso de informacao na primeira tela

## Regra De Tela

Mostrar somente o que importa para a acao operacional daquele momento.

Informacoes secundarias devem ficar em:

- Detalhes
- Abas
- Edicao
- Expansao

## Navegacao Principal

```text
COMERCIAL
- Projetos
- Clientes

PRODUTOS
- PN
- PDFs

ROTEIROS FABRICACAO
- Materiais
- Operacoes
- Tecnologias

ESTOQUE
- Saldos
- Movimentacoes

PRODUCAO
- OF
- OP
```

## Dashboard

Neste momento, nao priorizar dashboard executivo.

A pagina inicial deve priorizar navegacao operacional, atalhos claros e tabelas compactas.

Na pagina Comercial, evitar cards superiores de area. Manter somente:

- Nome do usuario
- Busca global
- Botao Novo Projeto
- Tabelas operacionais

## Tabelas

Priorizar tabelas operacionais em vez de excesso de cards.

As tabelas devem ter:

- Leitura rapida
- Espacamento equilibrado
- Densidade operacional
- Visual limpo

## Linguagem

Usar linguagem de fabrica:

- Projeto
- PN
- Roteiro Fabricacao
- Producao
- Saldo
- Movimentar
- Preparar material
- OF
- OP

Evitar termos tecnicos de TI na interface.
