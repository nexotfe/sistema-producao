# NEXOTFE

# MANIFESTO

**Versão:** 1.0
**Status:** Permanente

---

# O NEXOTFE nasceu da indústria.

O NEXOTFE não nasceu da tecnologia.

Nasceu da experiência prática em manufatura, usinagem, engenharia, suprimentos, PCP, produção e gestão industrial.

Foi criado para representar o funcionamento real de uma indústria, transformando conhecimento em uma ferramenta simples, integrada e confiável.

Nosso objetivo nunca foi desenvolver apenas mais um ERP.

Nosso objetivo é construir uma plataforma que ajude pessoas a tomar melhores decisões.

---

# O processo vem antes do software.

O software deve acompanhar o processo industrial.

O processo industrial nunca deverá ser alterado para atender limitações do software.

Antes de qualquer desenvolvimento é necessário compreender como a empresa trabalha.

Somente depois essa lógica será transformada em software.

---

# Simplicidade é um compromisso.

A simplicidade não significa fazer menos.

Significa eliminar tudo aquilo que não agrega valor.

Cada tela.

Cada botão.

Cada cadastro.

Cada relatório.

Cada processo.

Deve existir porque resolve um problema real.

Sempre que existir uma solução mais simples, mais clara e mais intuitiva, ela deverá ser considerada.

---

# Evolução permanente.

O NEXOTFE é um sistema vivo.

Nenhum documento desta Base de Conhecimento é definitivo.

Nenhuma regra é imutável.

Nenhuma funcionalidade é permanente.

O projeto deverá evoluir continuamente.

Sempre.

---

# O conhecimento é o maior patrimônio.

Tecnologias mudam.

Linguagens mudam.

Frameworks mudam.

O conhecimento permanece.

Por esse motivo, toda regra de negócio deverá nascer primeiro na Base de Conhecimento.

Somente depois será transformada em código.

O software é consequência do conhecimento.

Nunca o contrário.

---

# Integração acima da duplicidade.

Nenhum módulo existe isoladamente.

Toda informação possui uma origem.

Toda informação possui consumidores.

O sistema deverá evitar duplicidade de dados.

Reduzir retrabalho.

Compartilhar conhecimento entre todos os módulos.

Uma informação deve nascer apenas uma vez.

---

# O papel do NEXOTFE.

O NEXOTFE não substitui a experiência das pessoas.

Ele organiza informações.

Constrói cenários.

Apoia decisões.

A decisão continuará sendo humana.

---

# O Método Nexus.

O Nexus representa a forma como o NEXOTFE é construído.

Cada evolução deverá respeitar os seguintes princípios:

* simplicidade;
* clareza;
* elegância;
* integração;
* evolução contínua;
* menor retrabalho possível;
* reutilização;
* consistência.

O objetivo não é construir o maior ERP.

O objetivo é construir o ERP mais inteligente para quem trabalha na indústria.

---

# Como desenvolvemos.

Todo desenvolvimento seguirá esta sequência:

1. Compreender o processo industrial.
2. Discutir a regra de negócio.
3. Registrar na Base de Conhecimento.
4. Validar a lógica.
5. Projetar a arquitetura.
6. Implementar.
7. Testar.
8. Evoluir.

Nunca iniciar diretamente pelo código.

---

# Nosso compromisso.

Desenvolver somente funcionalidades que agreguem valor ao processo industrial.

Questionar sempre.

Simplificar sempre que possível.

Evoluir continuamente.

Preservar a lógica do negócio acima da tecnologia.

---

# A frase que orienta este projeto.

> **A tecnologia evolui.**
>
> **O conhecimento permanece.**
>
> **A simplicidade é um compromisso permanente.**
>
> **Esse é o NEXOTFE.**
